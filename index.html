03 Modul 3

Mengembangkan Aplikasi AI
dengan Azure AI Foundry SDK

🧰

Apa itu SDK?

  • Menyatukan berbagai layanan AI dalam satu akses programmatic
  • Mendukung Python, .NET, JavaScript + REST API
  • AIProjectClient — class utama koneksi ke proyek
  • Otentikasi via DefaultAzureCredential / az login
  • Sync & async operations didukung
🔗

Connections

  • Resource/Hub — Koneksi induk multi-proyek
  • Project — Koneksi spesifik proyek
  • connections.list() — Daftar semua koneksi
  • connections.get(name, include_credentials)
🤖

Kapabilitas Tambahan SDK

  • Agent Management — Buat, jalankan, dan versioning AI agents
  • Tools: Code Interpreter, Azure AI Search, Bing Grounding, File Search, Function Tool, Image Gen
  • Dataset Management — Upload dokumen, buat dataset
  • Search Index — Buat & enumerate search indexes
  • Evaluation — Red-team tests & evaluation rules
  • Fine-tuning — Model tuning dari kode
💬

Membangun AI Chat Client

Python
from azure.ai.projects import AIProjectClient
from azure.identity import DefaultAzureCredential

client = AIProjectClient(
    endpoint="https://<project>.api.azureml.ms",
    credential=DefaultAzureCredential()
)
chat = client.get_openai_client()
response = chat.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "Kamu adalah asisten."},
        {"role": "user", "content": prompt}
    ]
)
print(response.choices[0].message.content)

Fine-tuning Model
Azure AI Foundry

🎯

RAG vs Fine-tuning

RAGFine-tuning
TujuanData faktual real-timeKonsistensi gaya & format
DataEksternal, dinamisTraining dataset (JSONL)
Kapan?Data sering berubahBrand voice spesifik
TeknikIndex + retrievalSFT (LoRA otomatis)
📋

Data Percakapan (JSONL)

JSONL
{"messages": [
  {"role": "system", "content": "Aturan main..."},
  {"role": "user", "content": "Pertanyaan..."},
  {"role": "assistant", "content": "Jawaban ideal...",
   "weight": 1}
]}

weight=1 dilatih, weight=0 hanya konteks. Anonimkan data sensitif!

⚙️

Hyperparameter & Teknik

batch_size learning_rate_multiplier (0.5–2) n_epochs seed
  • LoRA — Low-Rank Adaptation, otomatis diterapkan untuk efisiensi
  • SFT — Supervised Fine-Tuning, cocok untuk domain specialization
  • RFT (baru) — Reinforcement Fine-Tuning untuk model terbaru
📊

Langkah & Evaluasi

  1. Pilih base model dari katalog
  2. Upload training + validation dataset
  3. Konfigurasi hyperparameter
  4. Submit job → Monitor loss function
  5. Evaluasi: Token Accuracy, relevance, semantic similarity
  6. Deploy & integrasikan via API

💡 Start small, iterate — pantau loss plateau, atur budget alerts

07 Modul 7

Responsible
Generative AI

M1

Map

Identifikasi & prioritasi bahaya: bias, halusinasi, konten ilegal. Red Team Testing dengan prompt ekstrem.

M2

Measure

Uji dengan prompt beragam. Buat baseline. Metrik evaluasi konsisten — manual & otomatis.

M3

Mitigate

4 layer: Model, Safety System, Grounding + System Message, UX Layer. Proses berkelanjutan.

M4

Manage

Legal, Privacy, Security review. Peluncuran bertahap, rollback plan, telemetry & audit trail.

🛡️

Fitur Azure AI Content Safety

Prompt Shields Groundedness Detection Protected Material Content Filters Custom Categories
  • Prompt Shields — Anti prompt injection (direct & indirect), fitur Spotlighting (2025) untuk deteksi input untrusted
  • Groundedness Detection — Cek akurasi output terhadap sumber, mode Reasoning & Non-Reasoning, auto-correction
  • Content Filters — Deteksi violence, hate, sexual, self-harm + multimodal analysis (2024)

Evaluasi Performa
Aplikasi Generative AI

📏

AI-Assisted Metrics

  • Coherence — Keterpaduan & logical flow
  • Relevance — Kesesuaian jawaban
  • Fluency — Kelancaran bahasa
  • Groundedness — Apakah output berbasis sumber?
  • GPT Similarity — Kesamaan semantik

GPT-4o sebagai "AI Judge" untuk scoring otomatis

🔢

NLP / Mathematical Metrics

  • BLEU — N-gram overlap (bilingual evaluation)
  • ROUGE — Recall-oriented (ROUGE-1, ROUGE-L)
  • METEOR — Synonym-aware matching
  • GLEU — Generalized BLEU
  • F1-Score — Precision × Recall balance

Tersedia via Azure AI Evaluation SDK (2024)

🛡️

Safety Evaluation

  • Hate & Unfairness detection
  • Violence content scoring
  • Sexual content detection
  • Self-harm risk scoring
  • Protected Material detection
  • Indirect Attacks evaluation
